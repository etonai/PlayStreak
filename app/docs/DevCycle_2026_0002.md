## Overview

This development cycle adds an Activity Detail view — analogous to the existing Piece detail view, but showing the full information for a single activity — and makes activities selectable from the Dashboard and Calendar tabs so users can open that detail view by tapping an activity.

## Current Behavior Reference

- **Piece detail pattern (the model for this cycle):** `PiecesFragment` shows a bottom-anchored `MaterialCardView` (`pieceDetailsCard` in `fragment_pieces.xml`) when a piece is selected, populated from `PiecesViewModel.selectedPieceDetails` LiveData, with a Close button to dismiss. Rows are conditionally visible.
- **Dashboard (`DashboardFragment`):** Today's and Yesterday's activities are rendered as plain text — all activities joined with newlines into a single `TextView` (`todayActivitiesList` / `yesterdayActivitiesList`). Individual activities are NOT currently tappable. The ViewModel already exposes them as `List<ActivityWithPiece>`.
- **Calendar (`CalendarFragment`):** Selected-date activities display in one of two modes depending on the "Calendar Activities Detail Mode" configuration setting:
  - *Simple mode:* one `TextView` (`selectedDateActivities`) with joined summary lines — not tappable.
  - *Detail mode:* per-activity Timeline-style cards (`item_timeline_activity.xml`) added to `linearLayoutSelectedDateActivities` via `populateActivityList()`/`bindActivityView()`, which already have Edit/Delete/Add buttons — but tapping the card body does nothing.
- **Data:** `ActivityWithPiece(activity, pieceOrTechnique)` already pairs each activity with its piece. The `Activity` entity has `timestamp`, `activityType`, `level`, `performanceType`, `minutes`, `notes`. The piece carries `name`, `type`, `isFavorite`, and (since DevCycle 2025-0021) `artist`, `key`, `notes`. **No database or repository changes are needed for this cycle.**

## Questions

**Q1: Detail view presentation — shared dialog or per-tab bottom card?**
The Piece detail is an inline bottom card inside `fragment_pieces.xml`. Replicating that inline approach would require duplicating the card and its wiring in both `fragment_dashboard.xml` and `fragment_calendar.xml`.
**Recommendation:** One shared `ActivityDetailDialogFragment` styled like the piece detail card — single implementation, identical behavior from both tabs.

**Q2: Should the detail view include Edit/Delete actions?**
Calendar detail mode cards already offer Edit/Delete; the Dashboard currently offers neither.
**Recommendation:** View-only with a Close button for this cycle (matches the Piece detail card, which is also view-only). Edit/Delete from the detail view can be a follow-up cycle if wanted.

**Q3: Should the Timeline tab also open the detail on tap?**
Out of the stated scope (Dashboard and Calendar), but the Timeline shows the same cards as Calendar detail mode.
**Recommendation:** Defer — keep this cycle scoped to Dashboard and Calendar; note Timeline as a future-cycle candidate.
