# Development Cycle 2026-0001

**Status:** 🎫 Open
**Start Date:** 2026-06-10
**Target Completion:** TBD
**Focus:** Practice Notes, Quick Save, and Always-On Pro Mode

## Overview

This development cycle focuses on three improvements to the activity logging experience: (1) allowing users to add notes to practice activities the same way they can for performance activities, (2) adding a Quick Save button to the Select Level screen that saves the activity immediately and skips the remaining screens, and (3) removing the Pro Mode toggle from the Configuration screen and making all users Pro by default.

## Current Behavior Reference

The add-activity flow (`ui/addactivity/`) currently navigates as follows:

- **Practice (piece, level 1, 3, 4):** Select Piece → Select Level → Summary
- **Practice (level 2 or any technique):** Select Piece → Select Level → Time Input → Summary
- **Performance:** Select Piece → Select Level (incl. performance type) → Notes Input → Summary

Key existing facts that simplify this cycle:

- The `Activity` entity already has a `notes` field used by all activity types; `AddActivityViewModel.saveActivity()` already persists it and `SummaryFragment` already displays it. **No database changes are needed for practice notes — this is purely a navigation/flow change.**
- `NotesInputFragment` already exists with Continue and Skip buttons and edit-mode pre-population; it just isn't reachable from the practice flow.
- The `summaryFragment` nav destination already defaults `minutes` to `-1` ("not recorded") and `notes` to `""`.
- `AddActivityViewModel.saveActivity(pieceId, activityType, level, performanceType, minutes, notes)` (the no-timestamp overload) already handles the pre-populated calendar date and activity limits — Quick Save can call it directly.
- The Pro Mode toggle was added in DevCycle 2025-0021 Phase 5 (`fragment_configuration.xml` + `ConfigurationFragment.setupProModeToggle()`); Phase 3 of this cycle supersedes that feature.

## Questions

**Q1: Quick Save behavior in edit mode?**
The Select Level screen is also used when editing an existing activity (via `EditActivityStorage`). Should QUICK SAVE be (a) hidden in edit mode, or (b) update the activity keeping its existing minutes/notes/timestamp and applying only the new level/performance type?
**Recommendation:** (a) Hide the button in edit mode — simplest, and avoids silently discarding existing time/notes data if implemented as a plain save.

**Q2: Remove the now-inert ProUserManager setters?**
`setProUser()`, `toggleProStatus()`, and `resetToFreeUser()` will have no effect on `isProUser()` once it returns true unconditionally. Remove them, or keep them for possible future re-monetization?
**Recommendation:** Keep them (and the SharedPreferences plumbing) — low cost, preserves the option to reintroduce tiers later.

**Q3: Notes screen title for practice?**
`notesInputFragment`'s nav label is "Add Notes" and is generic. Is the existing screen text acceptable for practice activities, or should the heading distinguish practice vs. performance notes?
**Recommendation:** Reuse as-is unless the screen contains performance-specific wording (verify during implementation).
